═══════════════════════════════════════════════════════════════════════════════
🚀 3-STEP COMPLETION PROCESS:
═══════════════════════════════════════════════════════════════════════════════

STEP 1: Get Railway Public Domain (2 min)
────────────────────────────────────────────────────────────────────────────────
Action: Find your Railway database's public domain
Where: https://railway.app/dashboard → MySQL Database → Public Domain
What it looks like: gateway.railway.app or mysql-prod-xxxx.railway.app
Why: Render needs a public domain to connect (private won't work)

STEP 2: Update Render (5 min)
────────────────────────────────────────────────────────────────────────────────
Action: Set DB_HOST to your public domain
Where: https://dashboard.render.com → school-management-backend-gnav → Environment
What to change: DB_HOST from ${{RAILWAY_PRIVATE_DOMAIN}} to your public domain
Then: Click Redeploy and wait 2-3 minutes
Verify: Logs should show "Connected to MySQL database"

STEP 3: Update Vercel (2 min)
────────────────────────────────────────────────────────────────────────────────
Action: Add API URL environment variable
Where: https://vercel.com/dashboard → school-management-system → Settings
What to add: VITE_API_URL = https://school-management-backend-gnav.onrender.com
Then: Click Save and Redeploy

🔑 CRITICAL INFORMATION:
═══════════════════════════════════════════════════════════════════════════════

What Changed:
  ❌ OLD: DB_HOST = ${{RAILWAY_PRIVATE_DOMAIN}}
  ✅ NEW: DB_HOST = gateway.railway.app (your actual public domain)

Why It Changed:
  Private domains only work within Railway's network
  Render is a different platform and can't access private domains
  Public domains work from anywhere (including Render)

Where to Find It:
  1. Go to: https://railway.app/dashboard
  2. Click your MySQL database
  3. Find "Public Domain" or "TCP Proxy Domain"
  4. Copy it exactly

🆘 TROUBLESHOOTING:
═══════════════════════════════════════════════════════════════════════════════

Problem: Backend shows ECONNREFUSED error
Solution:
  1. Go to Render Environment Variables
  2. Check DB_HOST is your PUBLIC domain
  3. Make sure it's NOT ${{RAILWAY_PRIVATE_DOMAIN}}
  4. Click Redeploy again
  5. Wait 5 minutes

Problem: Frontend shows "Failed to fetch" error
Solution:
  1. Go to Vercel Environment Variables
  2. Verify VITE_API_URL is set
  3. Click Redeploy
  4. Hard refresh browser (Ctrl+Shift+R)
  5. Try logging in again

Problem: Can't find Railway public domain
Solution:
  → Read: 📍_FIND_RAILWAY_PUBLIC_DOMAIN.txt
