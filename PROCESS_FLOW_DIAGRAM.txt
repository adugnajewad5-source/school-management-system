WHAT HAPPENS BEHIND THE SCENES:
═════════════════════════════════════════════════════════════════════════════

When user runs: git push origin main

  1. Git connects to GitHub
  2. Sends changes (vercel.json update)
  3. GitHub receives changes
  4. GitHub triggers Vercel webhook
  5. Vercel receives notification
  6. Vercel clones latest repository
  7. Vercel reads vercel.json
  8. Vercel runs: npm --prefix frontend ci
     - Cleans node_modules
     - Installs exact versions from package-lock.json
     - More reliable than npm install
  9. Vercel runs: npm --prefix frontend run build
     - Compiles React code
     - Bundles JavaScript
     - Optimizes assets
  10. Vercel uploads to CDN
  11. Vercel updates DNS
  12. Frontend URL now serves new build
  13. SPA routing works correctly
  14. /register route accessible

TROUBLESHOOTING:
═════════════════════════════════════════════════════════════════════════════

If git push fails:
  → Check GitHub credentials
  → Use Personal Access Token if needed
  → Verify internet connection

If Vercel deployment fails:
  → Check Vercel dashboard for build logs
  → Verify environment variables are set
  → Try manual redeploy from Vercel dashboard

If register page still shows 404:
  → Hard refresh browser (Ctrl+Shift+R)
  → Clear browser cache
  → Try in incognito/private window
  → Check Vercel deployment status
