STEP 1: DOWNLOAD GITHUB DESKTOP
═════════════════════════════════════════════════════════════════════════════

1. Open your browser
2. Go to: https://desktop.github.com/
3. Click the big blue "Download for Windows" button
4. Wait for download to complete
5. Run the installer (GitHubDesktopSetup.exe)
6. Follow the installation wizard
7. GitHub Desktop will open automatically

Expected: GitHub Desktop window opens


STEP 2: SIGN IN TO GITHUB
═════════════════════════════════════════════════════════════════════════════

When GitHub Desktop opens:

1. You'll see a "Welcome to GitHub Desktop" screen
2. Click "Sign in with your browser"
3. Your browser will open GitHub login page
4. Enter your GitHub username
5. Enter your GitHub password
6. Click "Sign in"
7. GitHub will ask to authorize GitHub Desktop
8. Click "Authorize desktop"
9. Return to GitHub Desktop
10. You're now signed in ✅

Expected: GitHub Desktop shows your GitHub account

TROUBLESHOOTING
═════════════════════════════════════════════════════════════════════════════

Problem: "Repository not found"
Solution:
  1. Make sure you're signed in to GitHub
  2. Try searching again
  3. If still not found, you might need to fork the repository first

Problem: "Authentication failed"
Solution:
  1. Click File → Options
  2. Click Accounts
  3. Sign out and sign in again
  4. Follow browser authentication

Problem: "Changes don't show up"
Solution:
  1. Make sure you're in the correct repository
  2. Click Repository → Show in Explorer
  3. Verify vercel.json is in the root directory

Problem: "Push failed"
Solution:
  1. Make sure you have internet connection
  2. Try pushing again
  3. Check GitHub status: https://www.githubstatus.com/

Problem: "Register page still shows 404"
Solution:
  1. Hard refresh browser: Ctrl+Shift+R
  2. Try in incognito/private window
  3. Check Vercel dashboard for deployment errors
  4. Wait a few more minutes for deployment to complete
