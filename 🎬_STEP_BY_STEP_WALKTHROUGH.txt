PART 1: SET RAILWAY DATABASE ON RENDER (5 MINUTES)
═══════════════════════════════════════════════════════════════════════════════

STEP 1: Open Render Dashboard
────────────────────────────────────────────────────────────────────────────────
1. Open your browser
2. Go to: https://dashboard.render.com
3. You should see your services listed
4. Look for: "school-management-backend-gnav"
5. Click on it

STEP 2: Navigate to Environment Variables
────────────────────────────────────────────────────────────────────────────────
1. You're now on your backend service page
2. Look at the left sidebar
3. Find and click: "Environment"
4. You should see a section called "Environment Variables"

STEP 3: Add First Variable - DB_HOST
────────────────────────────────────────────────────────────────────────────────
1. Click: "Add Environment Variable" button
2. In the "Key" field, type: DB_HOST
3. In the "Value" field, type: ${{RAILWAY_PRIVATE_DOMAIN}}
   ⚠️  IMPORTANT: Make sure you type the double curly braces: ${{
4. Click: "Save"
5. Wait for it to save (you'll see a confirmation)

STEP 4: Add Second Variable - DB_USER
────────────────────────────────────────────────────────────────────────────────
1. Click: "Add Environment Variable" button again
2. In the "Key" field, type: DB_USER
3. In the "Value" field, type: root
4. Click: "Save"

STEP 5: Add Third Variable - DB_PASSWORD
────────────────────────────────────────────────────────────────────────────────
1. Click: "Add Environment Variable" button again
2. In the "Key" field, type: DB_PASSWORD
3. In the "Value" field, type: oGGYFZAYYVfJyMReLooWEFXCiWETNGep
   ⚠️  IMPORTANT: Copy this EXACTLY - it's case-sensitive!
4. Click: "Save"

STEP 6: Add Fourth Variable - DB_NAME
────────────────────────────────────────────────────────────────────────────────
1. Click: "Add Environment Variable" button again
2. In the "Key" field, type: DB_NAME
3. In the "Value" field, type: railway
4. Click: "Save"

STEP 7: Add Fifth Variable - DB_PORT
────────────────────────────────────────────────────────────────────────────────
1. Click: "Add Environment Variable" button again
2. In the "Key" field, type: DB_PORT
3. In the "Value" field, type: 3306
4. Click: "Save"

STEP 8: Add Sixth Variable - NODE_ENV
────────────────────────────────────────────────────────────────────────────────
1. Click: "Add Environment Variable" button again
2. In the "Key" field, type: NODE_ENV
3. In the "Value" field, type: production
4. Click: "Save"

STEP 9: Add Seventh Variable - JWT_SECRET
────────────────────────────────────────────────────────────────────────────────
1. Click: "Add Environment Variable" button again
2. In the "Key" field, type: JWT_SECRET
3. In the "Value" field, type: your_super_secret_jwt_key_123_change_in_production
4. Click: "Save"

STEP 10: Add Eighth Variable - FRONTEND_URL
────────────────────────────────────────────────────────────────────────────────
1. Click: "Add Environment Variable" button again
2. In the "Key" field, type: FRONTEND_URL
3. In the "Value" field, type: https://school-management-system.vercel.app
4. Click: "Save"

STEP 11: Verify All Variables Are Added
────────────────────────────────────────────────────────────────────────────────
1. Scroll down to see the "Existing Variables" section
2. You should see all 8 variables listed:
   ✓ DB_HOST
   ✓ DB_USER
   ✓ DB_PASSWORD
   ✓ DB_NAME
   ✓ DB_PORT
   ✓ NODE_ENV
   ✓ JWT_SECRET
   ✓ FRONTEND_URL

STEP 12: Redeploy Backend
────────────────────────────────────────────────────────────────────────────────
1. Scroll to the top of the page
2. Look for a button that says "Redeploy" or "Manual Deploy"
3. Click it
4. You should see a message like "Deployment started"
5. Wait 2-3 minutes for deployment to complete

STEP 13: Check Logs
────────────────────────────────────────────────────────────────────────────────
1. Click on "Logs" in the left sidebar
2. Watch the logs scroll by
3. Wait for the message: "Connected to MySQL database"
4. If you see this message, ✅ SUCCESS! Backend is now connected!
5. If you see error messages, check the troubleshooting section

PART 2: SET API URL ON VERCEL (3 MINUTES)
═══════════════════════════════════════════════════════════════════════════════

STEP 1: Open Vercel Dashboard
────────────────────────────────────────────────────────────────────────────────
1. Open your browser
2. Go to: https://vercel.com/dashboard
3. You should see your projects listed
4. Look for: "school-management-system"
5. Click on it

STEP 2: Navigate to Environment Variables
────────────────────────────────────────────────────────────────────────────────
1. You're now on your project page
2. Click: "Settings" (in the top navigation)
3. In the left sidebar, click: "Environment Variables"
4. You should see a section to add variables

STEP 3: Add API URL Variable
────────────────────────────────────────────────────────────────────────────────
1. Click: "Add New" or "Add Environment Variable" button
2. In the "Name" field, type: VITE_API_URL
3. In the "Value" field, type: https://school-management-backend-gnav.onrender.com
4. Under "Select Environments", make sure "Production" is checked
5. Click: "Save" or "Add"

STEP 4: Redeploy Frontend
────────────────────────────────────────────────────────────────────────────────
1. Go to "Deployments" tab (top navigation)
2. Find the latest deployment
3. Click the three dots (...) next to it
4. Click: "Redeploy"
5. Wait 2-3 minutes for deployment to complete
